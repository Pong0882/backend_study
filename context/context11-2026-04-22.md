### Issue #48 — 일봉 테이블 페이지네이션 + 차트 엔드포인트 분리 ✅

**원인 분석:**
- 렉 원인이 서버가 아닌 브라우저 렌더링임을 확인
- Network 탭: 서버 응답 281ms / 59KB → 빠름
- Performance 탭: Main thread 822ms 블로킹
- 차트 주석처리 테스트 → 렉 사라짐 → 차트 destroy/재생성 + DOM 18,000개 노드 생성이 원인

**구현:**
- `StockPriceRepository` — `Page<StockPrice> findByStockOrderByTradeDateDesc(Stock, Pageable)` 추가
- `StockService` — `getDailyPrices()` 페이지네이션 / `getAllDailyPrices()` 차트용 분리
- `StockController` — `/prices?page=0&size=50` + `/prices/chart` 엔드포인트 분리
- `Stocks.jsx` — `chartPrices`/`prices` state 분리, 테이블 페이지 버튼 추가

**결과:**
- 테이블 응답: 59.2KB → 7.9KB (-87%), 281ms → 26ms (-91%)
- 종목 변경 시 렉 사라짐

### Issue #49 — 증권사 계좌 등록 시 KIS API 유효성 검증 (진행 중 🔧)

- `docker-compose.yml`에 `ENCRYPT_SECRET_KEY: ${ENCRYPT_SECRET_KEY}` 추가
- 커밋/배포 완료

**구현 (빌드/테스트 전):**
- `ErrorCode.BROKER_ACCOUNT_INVALID_KEY` 추가 (400 Bad Request)
- `BrokerAccountService.validateKisApiKey()` — DB 저장 전 KIS `/oauth2/tokenP` 호출로 키 유효성 검증
  - MOCK → `https://openapivts.koreainvestment.com:29443`
  - REAL → `https://openapi.koreainvestment.com:9443`
  - 검증용 토큰은 저장하지 않음 (KisAuthService 캐시와 무관)

**남은 것:**
- `./gradlew build` 빌드 확인
- Docker 올려서 Swagger 테스트
  - 유효하지 않은 키 → 400 확인
  - 유효한 키 → 201 확인
- 커밋/푸쉬/PR/이슈 닫기
