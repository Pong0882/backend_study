## 오늘 한 것

### Issue #34 — KIS 주문 전송 EGW00356 / EGW00202 원인 파악 및 해결

**EGW00356 원인:**
- `KisAuthServiceTest`에 실전 서버 URL(`openapi.koreainvestment.com:9443`) 하드코딩
- 실전 서버로 토큰 발급 → 모의 서버로 주문 전송 → 서버 불일치 → `EGW00356`

**EGW00202 원인:**
- Spring RestClient에서 `Map`을 body로 넘기면 `Transfer-Encoding: chunked`로 전송됨
- KIS 게이트웨이가 chunked를 처리하지 못해 라우팅 단계에서 거부

**구조 개선:**
- `KisConfig` — `mockBaseUrl` + `realBaseUrl` 분리 (appKey/appSecret은 공용 시세 조회용으로 유지)
- `KisAuthService.getAccessToken()` — 공용 (시세 조회, Redis key: `kis:token:common`)
- `KisAuthService.getAccessToken(BrokerAccount)` — 계좌별 (주문, Redis key: `kis:token:{accountId}`)
- `KisOrderService` — `Map` → `byte[]` 직렬화로 `Content-Length` 명시
- `KisAuthService` — `EGW00133` rate limit 예외 처리 추가
- `KisAuthServiceTest` — `BrokerAccount` mock 기반으로 재작성

**현재 결과:**
```
[KisOrder] 주문 실패 — 모의투자 장종료 입니다.
```
게이트웨이 통과 확인. 장 중 테스트만 남음.
