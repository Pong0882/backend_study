# Linux 텍스트 에디터 비교 — nano vs vim

## nano

- 초보자 친화적. 화면 하단에 단축키가 항상 표시됨
- 별도 모드 없음 — 열자마자 바로 입력 가능
- 기능이 단순해서 복잡한 편집엔 부족

```bash
nano 파일명

# 저장: Ctrl+O → Enter
# 종료: Ctrl+X
# 저장 후 종료: Ctrl+X → Y → Enter
```

## vim

- 강력하지만 처음엔 어려움 — 모드 개념을 이해해야 함
- 숙달되면 키보드만으로 빠른 편집 가능
- 서버 어디에나 설치돼 있음 (vi는 거의 모든 Unix/Linux에 기본 탑재)

### vim 모드

| 모드 | 진입 방법 | 용도 |
|------|----------|------|
| Normal 모드 | 기본 / `Esc` | 커서 이동, 명령 입력 |
| Insert 모드 | `i` | 텍스트 입력 |
| Visual 모드 | `v` | 텍스트 선택 |
| Command 모드 | `:` | 저장, 종료 등 |

### 자주 쓰는 명령어

```bash
vim 파일명

i          # Insert 모드 진입 (입력 시작)
Esc        # Normal 모드로 복귀

:w         # 저장
:q         # 종료
:wq        # 저장 후 종료
:q!        # 저장 없이 강제 종료

dd         # 현재 줄 삭제
yy         # 현재 줄 복사
p          # 붙여넣기
u          # 실행 취소 (undo)
/검색어    # 검색
```

## 언제 무엇을 쓰나

| 상황 | 추천 |
|------|------|
| 빠르게 설정 파일 한 줄 수정 | nano |
| 복잡한 편집, 대용량 파일 | vim |
| 서버에 nano 없을 때 | vim (항상 있음) |
