# Harness Testing (AI 모델 평가 프레임워크)

## Harness Testing이란

AI 모델의 품질과 안전성을 체계적으로 평가하기 위한 테스트 프레임워크.

일반 소프트웨어 테스트(JUnit 등)가 "코드가 의도대로 동작하는가"를 검증한다면, Harness Testing은 **"AI 모델이 다양한 입력에 대해 올바르고 안전하게 응답하는가"**를 검증한다.

---

## 왜 필요한가

AI 모델은 일반 코드와 달리 동일한 입력에도 출력이 달라질 수 있고, 예상치 못한 방식으로 동작할 수 있다.

```
일반 코드 테스트
    input: 1 + 1
    expected: 2
    → 항상 동일한 결과

AI 모델 테스트
    input: "삼성전자 지금 사야 해?"
    expected: 적절한 면책 고지 포함, 투자 권유 없음, 할루시네이션 없음
    → 매번 다른 응답, 다양한 기준으로 평가 필요
```

---

## 주요 평가 항목

| 항목 | 설명 |
|------|------|
| 정확성 | 사실과 다른 내용(할루시네이션)을 생성하는가 |
| 안전성 | 유해한 콘텐츠, 개인정보, 민감 정보를 출력하는가 |
| 일관성 | 동일한 질문에 일관된 답변을 하는가 |
| 거부 능력 | 부적절한 요청을 올바르게 거부하는가 |
| Prompt Injection 저항성 | 악의적인 프롬프트로 동작을 조작할 수 있는가 |

---

## Claude Code 유출 문서에서 언급된 맥락

Anthropic이 Claude를 개발하면서 모델이 코드 작성, 도구 사용, 안전 정책 준수 등을 올바르게 수행하는지 검증하기 위해 Harness Testing을 사용한다고 알려짐.

특히 **agentic 환경** (AI가 도구를 직접 실행하는 환경)에서는 단순 텍스트 응답 검증을 넘어, 실제 도구 호출 결과까지 포함한 end-to-end 검증이 필요하기 때문에 중요도가 높다.

## pong-to-rich 연관성

Phase 21에서 AI Agent를 구현할 때 직접 관련된다.

```
뉴스 감성 분석 Agent
    → "삼성전자 관련 뉴스가 긍정적인가?"
    → 할루시네이션 없이 실제 뉴스 기반으로 답하는가?
    → Positive/Negative/Neutral 외 이상한 값 반환하지 않는가?

자동매매 신호 Agent
    → 매매 신호가 실제 전략 로직 기반인가?
    → 임의로 매수/매도를 권유하지 않는가?
```

실제 구현 시 AI 응답을 그대로 믿지 않고 **검증 레이어**를 두는 것이 Harness Testing 개념의 실무 적용이다.
