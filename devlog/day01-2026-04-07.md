## 깨달은 점 / 막혔던 것

- hello-spring/ 안에 .git 이 있어서 nested git repo 오류 발생 → `rm -rf hello-spring/.git` 으로 해결
- `git add .` 보다 파일/폴더 명시해서 add 하는 습관 들이기
- `.env` 파일은 docker-compose가 읽는 것이지 Spring이 자동으로 읽지 않음 → `application-local.yml`에 값 직접 입력해야 Spring이 인식
- 에러 로그:
  ```
  Access denied for user '${MYSQL_USER}'@'localhost' (using password: YES)
  ```
  → `${MYSQL_USER}` 가 환경변수로 치환되지 않고 문자열 그대로 MySQL에 전달된 것을 보고 Spring이 `.env`를 읽지 못하고 있다고 판단 → `application-local.yml`에 값 직접 입력으로 해결
- 로컬 MySQL 서비스(MySQL80)가 3306 점유 중이라 Docker 컨테이너 포트 충돌 → `net stop MySQL80` 으로 해결

![로컬 MySQL 서비스 정지 (net stop MySQL80)](https://github.com/user-attachments/assets/842033c4-7f0f-498b-adf9-e13b2bfc10e6)


- 도커 네트워크(pong-network) 구성 — 나중에 Redis, Kafka 등 추가 시 컨테이너끼리 이름으로 통신 가능
- devlog에 스크린샷 첨부 방법: git repo에 이미지 직접 올리면 용량 문제 발생 → GitHub Issues 본문에 이미지 드래그 앤 드롭하면 자동으로 링크 생성됨 (`https://github.com/user-attachments/assets/...`) → 그 링크를 md에 붙여넣기. Issue 저장 안 해도 링크는 영구 유효. 공식적으로 지원하는 방법.
