### Axios 인터셉터

모든 요청에 Access Token 자동 첨부, 401 응답 시 Refresh Token으로 재발급 후 원래 요청 재시도.

```js
// 요청 인터셉터 — Access Token 자동 첨부
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('accessToken')
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

// 응답 인터셉터 — 401 시 refresh 후 재시도
api.interceptors.response.use(
  (res) => res,
  async (err) => {
    if (err.response?.status === 401 && !original._retry) {
      // refresh token으로 재발급 → 원래 요청 재시도
    }
  }
)
```

### 버그 수정 — import_stock_csv.py 종목코드 앞자리 0 손실

**상황:** 시세 페이지에서 SK하이닉스 선택 시 데이터가 안 뜸

**에러 원문 (Spring 로그):**
```
[기간별시세] 1페이지 호출 — 20260407 ~ 20260417
[기간별시세] 반환 데이터 없음 — 종료
[기간별시세] 전체 완료 — 종목: 660, 총 저장: 0건
```

**첫 판단:** 종목코드가 `000660`이어야 하는데 `660`으로 전달되고 있음. DB에 잘못 저장된 것.

**원인 분석:** `import_stock_csv.py`에서 pandas가 CSV를 읽을 때 `stock_code` 컬럼을 숫자로 파싱. `000660` → `660`으로 앞자리 0 손실.

**해결:**
```python
# 수정 전
df = pd.read_csv(csv_path)

# 수정 후
df = pd.read_csv(csv_path, dtype={'stock_code': str})
```

**배운 점:** pandas `read_csv()`는 기본적으로 컬럼 타입을 자동 추론. 종목코드처럼 숫자처럼 생긴 문자열은 반드시 `dtype` 명시 필요.

**후속 조치 필요:** DB에 이미 잘못 들어간 데이터 재삽입 필요
```sql
DELETE FROM stock_prices;
DELETE FROM stocks;
```
→ `import_stock_csv.py` 재실행

### CORS 에러 — api.pongtrader.pro Swagger UI에서 API 호출 불가

**상황:** `https://api.pongtrader.pro/swagger-ui`에 접속해서 API를 호출하려 하니 에러 발생

**에러 원문:**
```
Failed to fetch
Possible Reasons: CORS, Network Failure,
URL scheme must be "http" or "https" for CORS request.
```

**첫 판단:** `curl`로 직접 치면 응답이 잘 오는데 Swagger에서만 안 됨. CORS는 브라우저 정책이니까 서버 응답 헤더 문제.

**시도한 것들:**
- `springdoc.server-url`이 `https://pongtrader.pro`로 설정되어 있어서 Swagger가 다른 도메인으로 API 호출하고 있었음 → `https://api.pongtrader.pro`로 변경

**원인 분석:**
1. `SecurityConfig`에 CORS 설정 자체가 없었음 — `Access-Control-Allow-Origin` 헤더가 응답에 없으니 브라우저가 차단
2. `springdoc.server-url`이 `pongtrader.pro`로 되어 있어서 Swagger UI가 다른 origin으로 요청을 날리고 있었음

**해결:**
```java
// SecurityConfig.java — CorsConfigurationSource 빈 추가
http.cors(cors -> cors.configurationSource(corsConfigurationSource()))

@Bean
public CorsConfigurationSource corsConfigurationSource() {
    CorsConfiguration config = new CorsConfiguration();
    config.setAllowedOrigins(List.of(
        "https://pongtrader.pro",
        "https://api.pongtrader.pro",
        "http://localhost:5173"
    ));
    config.setAllowedMethods(List.of("GET", "POST", "PUT", "DELETE", "PATCH", "OPTIONS"));
    config.setAllowedHeaders(List.of("*"));
    config.setAllowCredentials(true);
    // ...
}
```
```yaml
# application-prod.yml
springdoc:
  server-url: https://api.pongtrader.pro
```

**왜 그 방법인가:** `WebMvcConfigurer`로만 설정하면 Spring Security 필터가 먼저 요청을 막아버림. Security 필터 체인 안에서 `.cors()` 로 연결해야 Preflight(OPTIONS) 요청도 인증 없이 통과됨.

**배운 점:** CORS는 서버가 막는 게 아니라 브라우저가 막는 것. `curl`은 되고 브라우저만 안 되면 CORS 의심. `allowCredentials(true)` 사용 시 `*` 와일드카드 Origin 불가 — Spring이 startup 시점에 예외.
