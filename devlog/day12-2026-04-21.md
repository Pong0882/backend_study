## 구현 내용

### Swagger server-url 환경변수화

기존 `application-prod.yml`에 `springdoc.server-url: https://api.pongtrader.pro`가 하드코딩되어 있었음.
로컬에서 Docker로 백엔드 띄우면 Swagger가 실제 서버로 요청을 날려서 로컬 테스트가 불가능한 문제.

```yaml
# 변경 전
springdoc:
  server-url: https://api.pongtrader.pro

# 변경 후
springdoc:
  server-url: ${SPRINGDOC_SERVER_URL:https://api.pongtrader.pro}
```

`docker-compose.yml`에 `SPRINGDOC_SERVER_URL` 환경변수 추가. 로컬 `.env`에서 `http://localhost:8080`으로 오버라이드.

### 프론트엔드 Docker 환경 구성

Vite 개발 서버(`npm run dev`)는 로컬 전용. 배포 환경에서는 빌드 결과물을 Nginx로 서빙해야 함.

**Dockerfile (2단계 빌드):**
- 1단계: Node 이미지로 `npm run build` → `dist/` 생성
- 2단계: Nginx Alpine 이미지에 `dist/` 복사 → 정적 파일 서빙

빌드 시 `VITE_API_BASE_URL`을 ARG로 주입. `import.meta.env.VITE_API_BASE_URL`로 axios baseURL 결정.

```
로컬 Docker: VITE_API_BASE_URL=http://localhost:8080  (.env)
운영 서버:   VITE_API_BASE_URL=https://api.pongtrader.pro (.env.production)
```

**포트:**
- `npm run dev` → 5173 (Vite 개발 서버)
- `docker compose up` → 3000 (Nginx, `3000:80` 매핑)

### CORS localhost:3000 추가

Docker로 프론트 띄우면 origin이 `http://localhost:3000`이라 기존 `5173`만 허용하던 CORS 설정에서 막힘.
`SecurityConfig.java`에 `http://localhost:3000` 추가.

### pong-network external 전환 + compose 파일 분리

백엔드/프론트/Nginx를 각각 별도 `docker-compose.yml`로 관리.
나중에 프론트를 다른 서버로 이전할 때 `frontend/docker-compose.yml`만 가져가면 됨.

같은 Docker 네트워크를 여러 compose 파일이 공유하려면 external 네트워크 방식 필요.

```bash
# 최초 1회 네트워크 생성
docker network create pong-network
```

각 compose 파일에서:
```yaml
networks:
  pong-network:
    external: true
```

Nginx conf에서 컨테이너 간 통신 시 `localhost` 대신 서비스 이름으로 접근:
```nginx
proxy_pass http://frontend:80;   # 프론트 컨테이너
proxy_pass http://app:8080;      # 백엔드 컨테이너
```

## 깨달은 점 / 막혔던 것

### Git Bash에서 docker exec 경로 변환 문제

```bash
docker exec pong-to-rich-frontend ls /usr/share/nginx/html
# → ls: C:/Program Files/Git/usr/share/nginx/html: No such file or directory
```

Git Bash가 `/usr/...` 경로를 Windows 경로로 자동 변환해버림.
`//`를 앞에 붙이면 변환 막힘:

```bash
docker exec pong-to-rich-frontend ls //usr/share/nginx/html
```

### 프론트 접속 안 되는 문제 (MIME type 에러)

```
Failed to load module script: Expected a JavaScript-or-Wasm module script
but the server responded with a MIME type of "text/html".
```

Nginx SPA fallback 설정 자체는 맞았는데 브라우저에 Service Worker가 캐싱되어 있어서 발생.
`Ctrl + Shift + R` 강력 새로고침으로 해결. 또는 개발자도구 → Application → Service Workers → Unregister.
